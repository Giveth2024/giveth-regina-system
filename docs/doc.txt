Endpoints 
1. Registration
POST /register

Description
Registers a new user account. The system validates the request, checks if the username or email already exists, hashes the password using bcrypt, and stores the user in the database.

Processing Flow
- Receive the registration request.
- Validate that all required fields are provided.
- Check whether the username or email already exists.
- Generate a bcrypt salt.
- Hash the user's password.
- Generate a UUID for the user ID.
- Insert the new user into the database.
- Return a successful registration response.


Request Body
{
    "full_name": "John Doe",
    "username": "johndoe",
    "email": "john@example.com",
    "password": "Password123",
    "phone_number": "0771234567"
}

Success Response
Status Code: 201 Created

Response Body
{
    "success": true,
    "message": "Registration was successful",
    "data": {
        "result": {
            "affectedRows": 1
        }
    }
}

Error Responses
- Status Code: 400 Bad Request

Response Body
{
    "success": false,
    "message": "Please fill out all required fields!"
}

- Status Code: 409 Conflict

Response Body
{
    "success": false,
    "message": "Username or Email is already Registered!"
}

2. Authentication

Login

POST /login

Description
Authenticates an existing user using either their username or email address together with their password. The system validates the request, verifies the user's credentials, generates a JSON Web Token (JWT), stores it in a secure HTTP-only cookie, and returns the authenticated user's information.

Processing Flow

Receive the login request.
- Validate that the required fields (identifier and password) are provided and that no unexpected fields are included.
- Search for a user using the supplied username or email address.
- If no matching user is found, return an "Invalid credentials" response.
- Compare the supplied password with the stored bcrypt password hash.
- If the password is incorrect, return an "Invalid credentials" response.
- Generate a JWT containing the user's ID, username, and email.
- Sign the JWT using the application's secret key.
- Store the JWT in a secure HTTP-only cookie.
- Return a successful authentication response.

Request Body

json
{
    "identifier": "johndoe",
    "password": "Password123"
}


> Note:The identifier field accepts either the user's usernameor email address.

Success Response

Status Code:200 OK

Response Body

json
{
    "success": true,
    "message": "Login successful!!",
    "data": {
        "full_name": "John Doe"
    }
}


Authentication Cookie
Upon successful authentication, the server creates an HTTP-only cookie containing the signed JWT.

Cookie Properties:

Cookie Name: token
HTTP Only: true
Secure: true (Production only)
SameSite: Strict
Expiration: 24 Hours
